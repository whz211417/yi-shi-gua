# 个人就餐体验升级实施计划

**Goal:** 增加今日状态、行动结果卡、个人菜品关系、七日回顾与每日小笺，同时保持本地优先和推荐兼容。

1. 在 `assets/student-meal-model.js` 与测试中增加状态权重纯函数：预算、时间、饱腹、心情；测试状态只轻度影响排序且不造成空候选。
2. 在 `assets/app.js` 存储版本中加入 `mealRelations`，实现常吃/已吃过/想尝试的无损迁移、持久化与排序输入；添加旧存储回归测试。
3. 在 `index.html`/`app.js`/`style.css` 增加默认收起的今日状态控件，并将状态、卦象、天气、食录解释渲染到三层结果卡。
4. 在菜单按需编辑面板加入三种关系状态切换，保持启用开关独立。
5. 从近七日记录计算本地非医疗回顾，并在食录增加每日小笺样式。
6. 完整测试、旧存储、375px/桌面浏览器、Pages 公网验收并发布。
