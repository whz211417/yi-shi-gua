# 易食卦·古籍卷轴式视觉重构设计

**日期：** 2026-07-13  
**状态：** 已获用户同意，待用户审阅本设计文档后进入实施计划

## 目标

将现有偏极简的“易食卦”改为现代中式的“古籍卷轴式”体验：保留深墨、米白、朱砂的克制基调，让布局更均衡、文案更有文言气质，并用低频、具仪式感的“掐指推演”动效替代普通翻卡。

不改变推荐算法、本地存储、菜单管理或离线运行方式。

## 现状问题

用户提供的桌面截图显示：

1. Hero 标题字号过大、四行断裂，左侧视觉质量过重，右侧留白无叙事用途。
2. 页面主要只有深色纹理与大字，缺少能解释“易食卦”主题的东方图形语言。
3. 现有语句偏现代说明，未建立古籍式仪式感。
4. 手机端若沿用该标题比例，首屏会被标题占满并造成阅读负担。

## 视觉方向

### 构图

- 桌面首屏采用 **古籍页眉 + 主签叙事 + 卦象侧栏** 的不对称双栏。
- 左栏为节制的主标题与一句说明；右栏不是空白，而是低对比的六爻谱、卦序编号、篆刻式朱砂印和极淡云纹。
- Hero 标题固定为两行：`今日食何` / `请报一数`；中文标点不作为独立行。
- 起卦区域以“案头签条”式层次呈现：条件为横向签条，报数为主操作，起卦按钮作为朱砂主印。
- 今日食录延续为古籍页脚的三道条目；均衡提示作为小幅批注，不抢主结果。

### 颜色与材质

- 保持深墨背景、米白正文和少量朱砂。
- 新增极低对比的纸纤维、山水墨痕、云纹和回纹，仅作背景层；禁止黄纸、符咒堆砌或廉价算命网页视觉。
- 朱砂仅用于页眉小印、活动状态、起卦主按钮、餐签揭晓瞬间。

### 字体与文案

- 标题与卦辞使用系统可用的宋体/明朝体回退序列；正文保持雅黑/系统无衬线，以确保可读性。
- 操作保持清晰，但改为轻文言：
  - “先说说此刻” → “请陈其时”
  - “起餐卦” → “拈数起卦”
  - “就吃这个” → “此餐可取”
  - “换一卦” → “再演一卦”
  - “管理菜单” → “整饬食单”
  - “今日食录” → “今朝食录”
- 饮食理由仍用清晰现代汉语，保留“仅作日常提示，非医疗建议”。

## 掐指推演动效

起卦动作总时长约 1.1–1.4 秒，分为四段：

1. **定数（0–220ms）：** 数字微缩，主按钮出现朱砂压印反馈。
2. **掐指（220–620ms）：** 六道爻线按顺序显现，模拟由下至上推演；不使用真实手部图片或伪科学表述。
3. **凝墨（620–960ms）：** 中央淡墨团聚，周边云纹/卦线轻微呼吸。
4. **落印揭签（960–1400ms）：** 朱砂圆印短暂落下，餐签完成 3D 翻面，露出推荐结果。

动效使用 CSS transform/opacity，不依赖第三方库；`prefers-reduced-motion: reduce` 时关闭分段运动，只保留即时显示与状态提示。

## 响应式与手机使用

- 手机（<700px）采用单列；标题缩小为 `clamp(42px, 13vw, 62px)`，最多两行，避免截图中的四行大字。
- 卦象侧栏改为标题后的横向小谱和印章，避免挤压正文。
- 条件签条可横向折行，触控目标不小于 44px。
- 菜单管理抽屉保持全屏，关闭按钮固定可见。
- 网页保持静态离线文件；电脑可通过同一 Wi‑Fi 下的本地 HTTP 地址供手机访问。后续可选加 PWA 清单，支持“添加到主屏幕”。

## 验收标准

1. 桌面首屏标题仅两行，构图左右均衡，右侧有克制的古风叙事层。
2. 手机 375px 宽度无横向滚动，首屏标题与报数控件均可见、可操作。
3. 起卦包含“爻线递进—凝墨—落印—翻签”四段视觉反馈，减少动效设置下无连续运动。
4. 所有改动无外部依赖；推荐、本地菜单和本地记录不回归。
5. 使用真实 Chrome 分别在桌面与手机视口检查截图、控制台错误与主要操作流程。
