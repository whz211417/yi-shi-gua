# 食单目录重构设计

**日期：** 2026-07-15  
**状态：** 用户已确认视觉方向，待实施计划审阅

## 目标

将“整饬食单”从逐项展开的编辑表单重构为可快速浏览的菜系目录。用户应能直接看到中国菜与世界料理的完整入口，按菜系筛选并在紧凑名单中启用或停用餐品，而不必在数百个输入框中滚动。

同时修复已保存的旧菜单只保留 81 项、导致新增世界料理与分层模板不可见的问题。

## 体验原则

1. 首页起卦区保持不变，不出现菜系偏好或分类选择。
2. “整饬食单”是唯一的菜单管理入口。
3. 目录先于菜品，名单先于详情，编辑按需出现。
4. 不承诺本地供应。模板保持“待确认”语义，用户可自行确认、启用或编辑。
5. 所有分类均来自 `DEFAULT_CUISINE_TAXONOMY`，不创建临时或伪造类别。

## 信息架构

### 顶部范围切换

- 中国菜
- 世界料理
- 我的启用

切换范围立即更新左侧目录与右侧名单，不依赖用户先打开菜系下拉框。

范围是严格的数据边界：

- 选中“中国菜”时，左侧只能出现 21 个中国菜系；不得在列表底部、滚动区域或“下一组”中出现任何世界料理入口。
- 选中“世界料理”时，左侧只能出现 18 个世界料理菜系；不得混入中国菜。
- 选中“我的启用”时，按各餐品真实 `cuisineZone` 分组显示，但仍先显示范围标签，避免用户误以为范围已切换。

### 菜系目录

桌面端为左侧固定目录，手机端转为横向可滚动的分组标签。目录显示实际菜品数量，按以下组别组织：

- 中国菜：八大菜系、地域与风味、日常与特别饮食
- 世界料理：东亚、东南亚、南亚、欧洲、美洲、中东/地中海、跨地域

中国菜需完整可见：川、粤、湘、鲁、苏、浙、闽、徽、京津、东北、西北、云南、贵州、潮汕、客家、港式、台式、素食、清真风味、地方小吃、中式日常。

世界料理需完整可见：日料、韩餐、泰餐、越南菜、马来/新加坡风味、印尼菜、印度菜、意餐、法餐、西式简餐、墨西哥菜、美式餐、土耳其、中东风味、地中海料理、轻食沙拉、三明治汉堡、咖啡烘焙。

### 紧凑菜品名单

选择一个菜系后，右侧只显示该菜系的紧凑行列表。每行包含：

- 启用状态与单击切换操作
- 餐品名称
- 小类与可用餐别
- 待确认状态（如尚未确认供应地点）
- 进入编辑的显式入口

不在列表中直接渲染名称、来源、餐别、主食、蛋白、蔬菜、口味等整套输入控件。点击“编辑”或菜名后，才以单独的编辑抽屉/对话面板呈现完整表单。

### 辅助筛选

菜系由目录承担。名单顶部只保留：

- 全文搜索（菜名、小类、菜系）
- 大类
- 餐别
- 只看已启用
- 清除筛选

搜索和辅助筛选均仅作用于目录后的名单，不影响首页起卦或餐签推荐范围。

## 数据与迁移

### 默认目录扩容

现有 228 项模板是下限，而非上限。扩展后默认目录目标为至少 **940 项**有明确菜名与分类路径的模板：

- 八大菜系（川、粤、湘、鲁、苏、浙、闽、徽）每个至少 **40 道**具体餐品，共至少 **320 项**，默认启用；
- 其余 13 个中国菜系每个至少 **20 道**具体餐品，共至少 **260 项**，默认启用；
- 世界料理 18 个菜系每个至少 **20 道**具体餐品，共至少 **360 项**，默认停用；
- 中国菜合计至少 **580 项**；
- 每个菜系至少覆盖 4 个有意义的大类，例如热菜、主食/面点、汤羹、凉菜、小吃、甜品或饮品中的适用类别，不以同一路径的无意义变体凑数；
- 新增条目必须有语义明确的菜名、稳定 ID、真实分类路径；
- 不得用数组位置、重复分类路径或虚构元数据凑数；
- 饮品、甜品等补充品不得作为午餐或晚餐的独立餐签。

### 旧 localStorage 合并

旧的 81 项或其他持久化菜单不能阻止新目录出现。首次读取已保存菜单时：

1. 将旧菜单标准化为完整分类路径；
2. 以稳定模板 ID 和标准化菜名匹配已知模板，迁移用户修改过的启用状态与可编辑字段；
3. 对未被旧菜单覆盖的默认模板补入目录；
4. 保留没有模板对应项的用户自定义餐品；
5. 写入目录版本标记，防止每次刷新重复注入；
6. 不修改既有食录记录。

结果是：旧用户会看到完整更新后的目录和世界料理，同时其旧餐品与手工调整仍保留。

## 视觉语言

保留当前克制暗色中式调性，但消除大段表单与超长滚动：

- 左侧目录以细线与文字层级建立导航，不使用大卡片墙；
- 名单行密度适中，使用名称、微型元信息和状态切换；
- 启用状态以单一金色强调，未启用使用低对比灰；
- 编辑面板是临时焦点，不占据浏览页面；
- 桌面采用目录 + 名单两栏，窄屏改为单栏，不产生横向溢出；
- 所有可点击控制保持最小 44px 点击区，键盘可访问，状态摘要使用 `aria-live`。

## 验收标准

1. 已保存旧 81 项菜单的浏览器中，打开菜单后也能看到完整世界料理目录。
2. 中国菜和世界料理切换均可用，世界料理初始餐品显示待启用。
3. 中国菜与世界料理的范围是严格隔离的：选择中国菜时目录仅含 21 个中国菜系，选择世界料理时仅含 18 个世界菜系。
4. 所有 39 个菜系在对应范围目录中可找到，显示数量与实际过滤结果一致。
5. 每个菜系至少具有设计目标规定数量的、明确命名的小类餐品，且大类覆盖合理。
6. 菜品主视图为紧凑名单，不出现逐项展开详情表单。
7. 点击启用世界菜后，它会参与后续餐签；未启用时不参与。
8. 搜索、大类、餐别、已启用筛选可组合，清除后恢复当前范围完整名单。
9. 桌面与 375px 窄屏均无横向溢出，首页仍无菜系输入。
10. 通过数据迁移、筛选、推荐、键盘与浏览器回归测试，并完成 GitHub Pages 公网验收。
