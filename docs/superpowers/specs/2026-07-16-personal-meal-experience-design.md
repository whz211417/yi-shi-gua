# 易食卦个人决策与食录体验升级设计

**日期：** 2026-07-16  
**状态：** 用户确认范围，待实施计划审阅

## 范围

本轮将“易食卦”从一次性推荐工具提升为轻量的个人就餐决策与食录系统，完成以下五项：今日状态、可执行结果、个人偏好标记、食录回顾、每日小仪式。保持首页直接数起卦，不增加菜系偏好或复杂表单。

## 1. 今日状态：轻量、可跳过

起卦区在原有餐别/去处/天气下增加一个默认收起的“今日状态”折叠区，不打断直接起卦。

- 预算：省钱 / 正常 / 想改善；
- 时间：赶时间 / 正常 / 可以慢慢吃；
- 饱腹：想吃饱 / 正常 / 想清淡；
- 心情：想热乎 / 想重口 / 想吃点好的。

默认值均为“正常/不限”。这些数据只参与候选排序和解释，不作为硬限制，也不持久化到跨日用户画像。结果应展示哪些状态实际影响了排序。

## 2. 可执行结果卡

结果卡固定分为三个行动层级：

1. **先去哪 / 吃什么类型**：明确食堂窗口或外出菜系；
2. **优先找什么**：2 至 4 个具体菜建议或外出小类；
3. **找不到怎么办**：两个同类替代方案。

卦象、天气、今日状态与近期食录的解释单列为“为什么这样建议”，与行动信息分开。不得用占卜语言掩盖现实约束；所有通用食堂菜继续标为非当日供应承诺。

## 3. 个人菜品关系

菜单中的每个餐品可拥有独立的本地状态：

- `frequent`：常吃；
- `tried`：已吃过；
- `wish`：想尝试；
- 未标记。

状态存入现有个人 localStorage 记录中，仍仅属于当前浏览器。它们不改变菜品启用状态：启用决定是否进入候选，关系状态决定排序。

推荐规则：食堂适度偏好常吃与现实可得；外出可适度偏好想尝试；近期已确认过的同餐、同主食、同蛋白、同菜系继续避重。状态只能轻微加权，不得压过天气、去处或完整正餐约束。

## 4. 食录与周回顾

确认餐继续写入日食录，并额外保留所属餐型、去处和选择时的建议标题。食录区域增加“近七日回顾”折叠摘要：

- 食堂 / 外出次数；
- 高频菜系或餐型；
- 主食、汤食、炸物等的重复趋势；
- 3 条非医疗、非诊断的下周尝试建议。

示例："本周面食出现较多，下次可试一次米饭配热汤。" 不显示卡路里、体重、健康风险或治疗结论。

## 5. 每日小仪式

首页保留简洁：根据当天是否已有确认记录，显示以下之一：

- "今日一餐尚未落笔"；
- "午餐已记：……"；
- "三餐有录"。

每次确认后生成一张当天的小笺摘要，包含日期、餐别、餐名、卦名和极短食候提示；它只作为食录条目样式，不新增分享、社交或云端功能。

## 数据与兼容

扩展存储版本以保存 `mealRelations` 与兼容的扩展记录字段。旧本地菜单、1,039 项目录、旧食录与目录迁移必须无损读取。没有状态时全部按未标记处理；存储不可用时仅当页生效，不报错。

## 验收

1. 首页仍能不展开任何新增控件直接起卦；
2. 四类今日状态均可影响解释与排序，但不会造成无候选；
3. 结果卡清楚呈现三层行动信息；
4. 三种个人关系状态可编辑、持久化，且不改变启用开关；
5. 常吃/想尝试能产生可测试的轻微排序差异；
6. 七日回顾只输出非医疗提示；
7. 旧本地状态、确认/重试、食堂/外出推荐、目录与移动端无回归；
8. 完成 375px、桌面和 Pages 验收。
