# 易食卦「墨环落印」动效重设计

**日期：** 2026-07-13  
**状态：** 已确认方向，待用户审阅规格  
**范围：** 仅替换报数起卦的仪式动效；不改变菜单、推荐、存储、卦辞或已发布站点的 URL。

## 1. 背景与目标

现有“掐指一算”通过六条爻线依次亮起完成过渡。该语言容易被理解为加载条或普通占卜页面特效，和页面的古籍卷轴、案头落款氛围不协调。

新方案以一笔墨与一枚朱砂印作为核心意象：不模拟命理推演，不宣称预测能力；它只为“将一次日常选择落定”提供一个克制的娱乐性揭签过程。

成功标准：

- 不再出现六爻逐根点亮、转圈、粒子或显著 3D 翻转；
- 动效具有纸、墨、印的实体感，但不堆叠素材；
- 从点击到结果可读总时长为约 0.95–1.15 秒；
- 桌面和手机使用同一视觉叙事；
- 减少动效模式同步安全展示结果；
- 起卦、重试、确认、持久化和推荐结果完全保持原行为。

## 2. 视觉语言

- **墨环：** 一笔未闭合的椭圆墨痕，边缘并不均匀；不连续旋转。它从低对比、局部可见收束到完整可见，再消退为结果卡背景中的隐约纹理。
- **朱砂印：** 方形“食”字印，不以弹跳或爆炸方式出现；从约 8px 的垂直距离压落，伴随短暂的轻微纸面压印感。
- **餐签：** 结果内容不是翻转卡片。餐名、档口、理由从墨环中心以 8–12px 的上移和透明度过渡显现，随后静止。
- **色彩：** 沿用现有深墨、米白、朱砂红、低饱和铜色令牌；只使用 CSS 渐变、伪元素和文字，不添加外部图片、字体或依赖。

## 3. 时间线与状态

`data-cast-phase` 继续是可检查的状态接口，但阶段由 `count → ink → seal` 构成。

| 时间 | 阶段 | 可见变化 | 交互 |
|---|---:|---|---|
| 0–180ms | `count` | 数字输入与起卦按钮产生一次短促的按落/朱砂描边反馈 | 起卦、重试、确认均禁用 |
| 180–760ms | `ink` | 结果卡中墨环由不完整笔触收束为可读圆环 | 按钮继续禁用 |
| 760–1040ms | `seal` | “食”印轻落；结果文本同时从墨环中心平稳显现 | 按钮继续禁用 |
| 1040ms 后 | 无阶段 | 墨环淡化为低对比底纹，餐签稳定可读 | 控件恢复 |

结果仅在 `seal` 阶段开始时写入和展示，确保推荐值确定后才可见。不得以连续旋转、无限动画或声音填充等待。

## 4. 结构与实现边界

- `index.html`：如现有结果卡缺少可语义承载的墨环/印章节点，可仅增加装饰性 `aria-hidden="true"` 元素；不改变已有表单 ID、按钮 ID、可访问名称或菜单结构。
- `assets/app.js`：保留现有 `casting` 锁、计时器清理、`pagehide` 清理、降动效分支、推荐计算与持久化流程。将旧的 `finger` 阶段调度替换为 `ink` 阶段；在 `seal` 阶段调用既有 `showResult()`；完成后统一恢复按钮。
- `assets/style.css`：删除或停用六爻点亮效果；实现墨环与印章的有限时长动画。结果卡内容在正常动效中只在 `seal` 时可见；在 `prefers-reduced-motion` 下不等待，也不播放连续动画。
- 不引入 Canvas、第三方库、网络请求、图片资源或计时器外的新状态。

## 5. 无障碍与降级

- 装饰性墨环和印章必须 `aria-hidden`，结果仍通过现有 aria-live 文案公布；
- 按钮禁用期间保持正常焦点顺序，不转移焦点；
- `prefers-reduced-motion: reduce` 时直接展示餐签，不显示分阶段等待或持续性动画；
- 结果的最终文字对比度、可选择性和键盘操作不因动效改变。
